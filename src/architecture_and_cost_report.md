## 一、 雲端大數據架構效能比較 (期中專題比較 - 15%)

在處理 TDCS M06A（12週，數百萬筆原始旅次）的巨量資料時，不同的計算架構會帶來截然不同的效能與維護成本。以下針對傳統架構與 AWS 雲端託管服務進行深度對比：

### 1. 各類大數據處理解決方案對比表

| 評估維度 | 方案 A：傳統本地 PC / 筆電 | 方案 B：AWS Cloud9 (單一虛擬機) | 方案 C：AWS EMR (託管 Hadoop/Spark 集群) | 方案 D：AWS Glue ETL (無伺服器 Spark) | 方案 E：AWS CloudShell 串流 + Athena (本專案採用) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **運算架構** | 地端單機運算 | 雲端單機運算 (EC2) | 雲端分散式集群 (MapReduce) | Serverless 分散式運算 | **Serverless 輕量串流 + 分散式 SQL 引擎** |
| **記憶體/磁碟限制** | 受限於地端硬體，處理 12 週大檔案極易 Out of Memory。 | 易受限於儲存空間 (Cloud9 預設僅 8GB-10GB 磁碟)，下載大檔案會爆硬碟。 | 強大，可處理 PB 級資料，但啟動速度慢 (約 5-10 分鐘)。 | 強大且全託管，但每次啟動有 2-3 分鐘冷啟動時間。 | **極佳！直接內存串流，完全不寫入本地磁碟，毫無磁碟爆滿風險。** |
| **處理效能** | 慢。單執行緒下載與解壓，處理 12 週資料需數小時。 | 中等。受限於所選 EC2 規格，且受限於 AWS 網路下載速度。 | 極快。多節點並行計算。 | 極快。自動彈性擴展 Spark 節點。 | **極快！CloudShell 內部連線 S3 頻寬高達數 Gbps，串流解析只需數分鐘。Athena 查詢在數秒內完成。** |
| **收費模式與成本** | 免費，但耗費個人電腦電力與時間。 | 依 EC2 啟動時間計費 (約 $0.05/小時)。 | 昂貴。按小時收取 EC2 與 EMR 授權費 (易耗盡學生額度)。 | 較貴。按 DPU-hour 計費 (最低消費 10 分鐘，約 $0.07/次)。 | **極省！CloudShell 完全免費。Athena 按查詢掃描量計費 ($5/TB)。過濾後掃描量幾近於零，成本為 $0.00。** |
| **權限相容性** | 無 AWS 權限限制，但需要安裝設定 Python 環境。 | **AWS Academy 經常阻擋 Cloud9 建立。** | **AWS Academy 嚴格阻擋 EMR 權限。** | **AWS Academy 限制角色建立，容易報錯。** | **100% 相容！AWS Academy 完全開放，無痛執行。** |

### 2. AWS 整合應用之效能優化說明 (期中評分重點)
*   **解耦運算與儲存 (Decoupling Compute and Storage)**：
    傳統資料庫需要把資料載入到本機硬碟才能查詢。本專案將**儲存**託管在 **Amazon S3**（高可靠、低成本），而**運算**採用 **AWS CloudShell**（串流清洗）與 **Amazon Athena**（Serverless 即時查詢）。這種架構不需要維護任何資料庫伺服器，真正實現了「有查詢才付費，沒查詢零維護成本」的現代雲端大數據架構。
*   **分散式查詢優勢 (Distributed Query Engine)**：
    Amazon Athena 底層基於 Facebook 開源的 Presto 分散式 SQL 引擎。當我們在 Athena 貼上 SQL 查詢時，AWS 會在後台調用數百個運算節點並行掃描 S3 資料。即便資料量高達數十億筆，也能在 **3 到 5 秒內**回傳結果，這比傳統地端單機 MySQL 跑索引快上百倍。

### 2. AWS 帳單檢視與監控攻略 (AWS Cost Explorer)

為了向教授證明你的運維與省錢能力，你可以在報告中加入以下 AWS 商業省錢維運步驟：

#### 步驟 A：設定 AWS Budgets 預算警報 (防止額度意外被扣光)
學生帳戶只有 $100 美元額度，最怕因為寫錯 PySpark 程式碼或無限循環查詢導致額度爆表。
1. 登入 AWS Console，搜尋 **AWS Budgets**。
2. 點擊 **Create budget** ➡️ 選擇 **Cost budget**。
3. 設定 Monthly Budget 額度為 **$10 USD**。
4. 在 Alert threshold 設定：當實際花費達到預算的 **80% ($8 USD)** 時，自動發送 Email 警報到你的信箱。這能100%確保你的 Academy 帳號絕對不會因為意外欠費而被鎖定。

#### 步驟 B：使用 AWS Cost Explorer 分析費用趨勢
1. 進入 AWS 帳單面板 (Billing Console)，點擊左側的 **Cost Explorer**。
2. 將 Group by 設為 **Service** (服務項目)，即可用直觀的柱狀圖看出到底是 S3 還是 Athena 花了最多錢。
3. 在專案成果中，截圖你的 Cost Explorer 畫面，證明專案實作期間每日花費均控制在極低水平（通常小於 $0.01 美元），這將是全班最具說服力的 **Volume (Value & Cost)** 證明！

#### 步驟 C：導入 S3 生命週期管理 (S3 Lifecycle Policy)
原始的 M06A tar.gz 壓縮檔體積龐大且下載完後通常不需要再讀取。
1. 在 S3 控制台點擊你的 Bucket ➡️ 選擇 **Management** 標籤。
2. 建立 **Lifecycle rule**，設定規則：當原始 tar.gz 檔案上傳超過 **14 天**後，自動將其轉移到 **S3 Glacier Deep Archive** 儲存級別。
3. 這樣可以將儲存單價從 $0.023/GB 降低至 **$0.00099/GB** (直接省下 **95%** 的 raw data 儲存費！)。
